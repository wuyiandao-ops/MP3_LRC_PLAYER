# 📄 產品需求文件 (PRD) — HTML5 LRC 繁簡中文同步播放器

## 1. 產品定位與願景
本產品是一款基於網頁標準技術（HTML5 + JavaScript）的**輕量化隨身 MP3 語訊與 LRC 歌詞同步播放器**。主要研發目的是為了解決傳統桌面應用程式在高頻率計時、跨平台文字渲染與媒體時間軸跳轉時，容易產生的 UI 執行緒卡死、介面當機、音訊不同步，以及繁簡中文 ANSI 編碼相容性等痛點。透過標準網頁技術，達到免安裝、跨平台（支援行動裝置）、極致流暢且高容錯的精讀與聆聽體驗。

## 2. 目標用戶與核心痛點
* **目標用戶**：需要對照逐行文字/歌詞進行音訊聆聽、經典精讀（如經書、講記校對）、語言學習或內容語訊校對的使用者。
* **核心痛點**：
    * **播放進度跳轉不同步**：手動拖曳播放條（Seeking）快轉或倒退時，字幕無法秒速跟上對應時間，產生視覺延遲。
    * **高頻率操作容易當機**：桌面套件的單執行緒架構在處理頻繁計時刷新與 UI 滾動時，容易因資源搶奪而卡死。
    * **繁簡中文編碼亂碼**：歷史沉澱的舊式 `.lrc` 歌詞檔案多採用 ANSI 變體（台灣為 Big5，中國大陸為 GBK/GB18030），在現代主流瀏覽器（預設 UTF-8）中極易呈現 `\ufffd`（實心問號）或空心問號之亂碼。

## 3. 核心功能需求
1. **無伺服器純本地執行 (Local-First)**：用戶雙擊打開 HTML 檔案或透過靜態網頁（如 GitHub Pages）即可於瀏覽器操作。音訊與歌詞完全在本地端沙盒內處理，不需架設伺服器，亦不上傳任何雲端，保障隱私與讀取速度。
2. **置頂控制面板佈局 (Top-Pinned Control Panel)**：所有核心操作元件固定於視窗最上方，確保下方大面積歌詞區滾動時，控制功能不被遮擋。
3. **智慧型三軌編碼自動容錯（繁簡通吃）**：系統預設以 `UTF-8` 進行解碼。若偵測到文字中含有亂碼替代特徵符號（如 `\ufffd` 實心問號 或 空心問號），或解析出的歌詞陣列長度為 0，系統須自動依序嘗試 `Big5`（繁體中文 Windows 預設）與 `GBK`（簡體中文 Windows 預設）進行解碼，確保繁簡舊檔均能自動還原為正確文字。
4. **毫秒級進度條拖曳同步**：使用者手動拖曳進度條並釋放滑鼠的瞬間，歌詞高亮與檢視區域必須在 0.01 秒內完成重新比對，並精準定位。
5. **平滑自動捲動與視窗置中 (Smooth Auto-Scroll & Center)**：當前播放到的歌詞行必須自動保持在歌詞顯示區域的**正中央**，並搭配平滑（Smooth）的滾動動畫。
6. **雙向同步 - 反向點擊跳轉 (Click-to-Seek)**：使用者可以用滑鼠（或手機手指）點擊下方的任何一行歌詞，播放器必須立即將音訊進度跳轉至該行歌詞的對應時間點並開始播放。
7. **硬體加速變速功能**：提供 0.5x 至 2.0x 的語速調節（以 0.1x 為單位），變速時音質須保持平順、維持原音高（Pitch）不變。
8. **長篇文字效能維護**：針對數萬字以上的長篇文稿（如經書、講記），大量 DOM 元件渲染不得造成網頁滑動或滾動時的卡頓。

## 4. 非功能性需求
* **跨平台相容性**：全面支持 Windows、macOS、Android (Chrome) 與 iOS (Safari) 瀏覽器，介面須具備 RWD 響應式換行防遮擋設計。
* **零依賴性**：不使用任何第三方 JS 框架（如 Vue、React）或外部 CDN 字型，確保單一 HTML 檔案即可獨立運行。
